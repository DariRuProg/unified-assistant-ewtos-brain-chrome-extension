Chrome Extension + KI-Server — Strategie & Evaluation

Einheitlicher KI-Assistent fur tagliche Arbeit. Verbindet YouTube-Research, Obsidian-Vault-Chat und Multi-LLM-Support in einer Chrome Extension — mit eigenem Python-Server als Gehirn und optionaler Claude-Code-Integration uber das MCP-Protokoll.

Autor: ewtos.com  |  Mai 2026

01

Aktueller Stand — Was ist implementiert?

Vollstandig implementiert (end-to-end funktionsfahig)

Feature Was der Nutzer sieht Technisch
Sidepanel UI Tab-Navigation (Vault/Web/Code), Dark Mode, Connection-Status Chrome MV3, WebSocket-Bridge
YouTube Transcript URL eingeben, Transcript extrahieren DOM-Injection via Content Script
Bookmark Manager 3 Modi: Rechtsklick, Multi-Tab-Shortcut, Text-als-Notiz HTTP POST an /tools/bookmarks
Notes & Scratchpad Textarea mit Auto-Save, Datei-Import, Export Debounced POST an Server
Todos Checkbox-Toggle, Falligkeitsdatum, Overdue-Counter notes/todos.md
Playlist Manager Rechtsklick YouTube, Playlist wahlen, Master-Page anlegen + Auto-Summary wiki/ Markdown-Files + Claude API
Vault Chat Streaming-Chat, Claude navigiert Vault via Tool-Use SSE-Streaming, Anthropic API
Multi-LLM Backend Anthropic, OpenAI, Ollama, Mistral konfigurierbar llm_providers/ Abstraktion
MCP-Integration Claude Code kann Extension-Tools direkt nutzen mcp_server.py (Stdio)

Noch nicht implementiert (im UI als “bald” markiert)

  • Page-Scrape — Markdown-Export einer kompletten Webseite
  • SEO-Check — h1–h6-Struktur, meta description, canonical, title
  • Image-Analyse — alt-Text, dimensions, filesize
  • Color-Picker — Farbpipette aus dem DOM
  • Screenshot + Annotation — Canvas-Drawing mit Pfeilen, Formen, Markierungen
Architektur

Chrome Extension (MV3) ←WebSocket→ Python FastAPI-Server ←→ LLM-Backends / Obsidian Vault. Server = Gehirn, Extension = Gesicht. Alle Chat-States, Tool-Definitionen und Vault-Logik leben am Server.

Marktanalyse — Wettbewerber

Kein direkter Konkurrent fur diese Feature-Kombination gefunden (Stand Mai 2026).

Der Markt fur KI-Browser-Extensions ist fragmentiert: Einzelne Tools decken je eine Dimension gut ab, aber keine Losung kombiniert alle funf relevanten Dimensionen in einem zusammenhangenden Produkt.

Identifizierte Marktlucken

Marktlucke 1

Keine einheitliche Losung kombiniert alle funf Dimensionen: Chat, YouTube-Verarbeitung, Vault-Sync, lokale LLM-Unterstutzung und MCP-Integration.

Marktlucke 2

Local-first-Betrieb mit Ollama ist stark unterversorgt. DSGVO-Anforderungen in Europa schaffen wachsende Nachfrage nach Losungen ohne Cloud-Abhangigkeit.

Marktlucke 3

MCP-Integration existiert in keiner vergleichbaren Chrome Extension. Das Protokoll etabliert sich als Standard fur KI-Tool-Integration — First-Mover-Vorteil.

Marktlucke 4

Agentur- und Power-User-Workflows (SEO-Check, Page-Scrape, Screenshot-Annotation) werden in keinem bestehenden Tool konsequent abgebildet.

Marktlucke 5

Die YouTube-zu-Vault-Pipeline erfordert bei allen Wettbewerbern mehrere manuelle Schritte. EwtosBrain schliesst diesen Loop in einem Klick.

Webentwickler / Agenturen

04

Starken & Schwachen

Klick auf einen Quadranten, um Details aufzuklappen.

Starken (Strengths) ▾
  • Einzige Losung mit dieser Feature-Kombination auf dem Markt
  • MCP-Integration als Alleinstellungsmerkmal unter Chrome Extensions
  • Privacy-Option durch Ollama — DSGVO-konform ohne Cloud
  • Vault-Chat nach Karpathy-Methode ohne RAG-Overhead
  • Erweiterbare Server-Architektur: neue Tools zentral hinzufugen
  • Dual-Rolle: eigenstandige App und MCP-Tool-Box fur Claude Code
  • Granulares Permission-Modell fur Vault-Schreibzugriffe
Schwachen (Weaknesses) ▾
  • Komplexes Setup: Python, venv, .env, Server starten
  • Keine Standalone-Nutzung ohne laufenden Server
  • Sprint 3 Browser-Tools fehlen noch (SEO, Scrape, Screenshot)
  • Kein Cloud-Hosting oder Docker-Image verfugbar
  • Kein einfaches Onboarding fur technisch weniger versierte Nutzer
  • Abhangigkeit von externen LLM-API-Keys fur Vollbetrieb
Chancen (Opportunities) ▾
  • Obsidian-Community ist technikaffin und adoptionsfreudig
  • Local-LLM-Markt wachst durch DSGVO-Druck in Europa
  • MCP wird zum Standard fur KI-Tool-Integration
  • B2B-Agentur-Nische nach Sprint 3 erschliessbar
  • Open-Source-Veroffentlichung kann organisch Community aufbauen
  • Keine starke etablierte Konkurrenz in dieser Feature-Kombination
Risiken (Threats) ▾
  • Chrome-Policy-Anderungen konnen Extension-APIs einschranken (MV3-Entwicklung)
  • Grosse Player wie Sider AI konnten Vault-Integration nachbauen
  • Python-Server-Abhangigkeit schreckt Non-Developer ab
  • LLM-API-Kosten konnen bei intensiver Nutzung fur Nutzer unuberschaubar werden
  • YouTube konnte DOM-Struktur andern und Transcript-Tool brechen

06

Setup-Wizard — Konzept

Um die Onboarding-Hurde zu senken, wird ein Setup-Wizard als erster Bildschirm eingebaut. Er fuhrt neue Nutzer Schritt fur Schritt durch die Konfiguration — ohne Anfassen der Einstellungsseite. Der Wizard ist als eigene Chrome Extension Page implementiert (extension/setup/wizard.html) und wird beim Erststart automatisch geoffnet.

Wizard-Schritte

1
Willkommen
Kurze Einfuhrung was EwtosBrain ist und was fur den Betrieb benotigt wird: Python, pip, API-Key. Checklist mit verlinkten Anleitungen fur Nutzer die noch nichts installiert haben.
2
Server verbinden
WebSocket-URL eingeben (Standard: ws://localhost:8765). Live-Verbindungstest mit visuellem Feedback. Falls Fehler: direkter Link zur Installations-Anleitung.
3
LLM-Provider
Dropdown: Anthropic, OpenAI, Ollama oder Mistral wahlen. Je nach Auswahl werden bedingte Felder eingeblendet: API-Key (Anthropic/OpenAI/Mistral) oder Server-URL (Ollama). Testanfrage an den Provider um Validierung zu bestatigen.
4
Vault verknupfen
Obsidian-Vault-Pfad eingeben, Vault-Namen vergeben, optionale Schreibrechte per Checkbox aktivieren. Server validiert ob der Pfad existiert und lesbar ist.
5
Bereit
Zusammenfassung der Konfiguration, direkte Links zu den Features. Hinweis: Setup kann jederzeit in den Einstellungen angepasst werden.
Hinweis

Der interaktive Prototyp des Wizards ist unter docs/wizard-mockup.html verfugbar. Der Wizard wird in extension/setup/wizard.html implementiert. background.js pruft beim Start ob das Setup abgeschlossen ist und offnet die Wizard-Page bei Erstinstallation automatisch.

Kritische Lucke

Das komplexe Setup (Python + venv + .env + Server starten) ist die grosste Adoption-Hurde. Setup-Wizard und Docker-Image haben oberste Prioritat vor dem Launch, weil alle anderen Marketing-Massnahmen ohne ein einfaches Onboarding verpuffen.
